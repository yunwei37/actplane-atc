\title[\sys{}]{\sys{}: Programmable OS-Level Policy Enforcement for Agent Harnesses}

\begin{abstract}
AI agents increasingly run in production through \emph{harnesses}, the software around the LLM,
including an engine that enforces safety and effectiveness \emph{policies}, e.g., ``run tests before committing.''
% AI 智能体越来越多地通过 \emph{harness} 运行于生产环境，harness 是围绕 LLM 的软件层，包含执行安全与有效性\emph{策略}的引擎，例如"提交前运行测试"。
Enforcing these policies requires bridging a \emph{semantic gap}: policy intent is expressed in underspecified natural language, while enforcement must act on concrete \emph{system actions}, e.g., which test to run.
% 执行这些策略需要弥合\emph{语义鸿沟}：策略意图以欠规范的自然语言表达，而执行层必须对具体的\emph{系统操作}做出判断，例如运行哪个测试。
Many policies also span multiple system events.
% 许多策略还跨越多个系统事件。
Yet existing approaches fall short. Tool-call guardrails miss system actions that bypass the tool layer, while OS sandboxes control resource access instead of actions, returning opaque errors that confuse the agent.
% 但是现有方法存在不足。工具调用护栏会遗漏绕过工具层的系统操作，而操作系统沙箱控制资源访问而非操作，返回的不透明错误令智能体困惑。
Our key insight is that policy context lives within the agent closest to the task, while enforcement must happen at the OS to cover all execution paths.
% 我们的关键洞察是，策略上下文存在于最接近任务的智能体中，而执行必须在操作系统层进行以覆盖所有执行路径。
We introduce \sys{}, a policy engine that lets agents declare policies and enforces them in the OS kernel with semantic feedback and isolation.
% 我们引入 \sys{}，一个策略引擎：智能体声明策略，内核以语义反馈与隔离执行。
\sys{} uses a simple information-flow control (IFC) DSL to support cross-event policies.
% \sys{} 使用简单的信息流控制（IFC）DSL 来支持跨事件策略。
We implement \sys{} with eBPF and evaluate it on policies from the empirical study, coding-task benchmarks, and safety benchmarks.
% 我们使用 eBPF 实现 \sys{}，并在来自实证研究的策略、编码任务基准和安全基准上进行评估。
\sys{} improves policy compliance, including on indirect execution paths that tool-call interception cannot observe, with 1.9\%--8.4\% overhead.
% \sys{} 提升了策略合规性，包括在工具调用拦截无法观察的间接执行路径上，开销为 1.9\%--8.4\%。
\sys{} is at \url{https://github.com/eunomia-bpf/ActPlane}.
% \sys{} 开源于 \url{https://github.com/eunomia-bpf/ActPlane}。
\end{abstract}
